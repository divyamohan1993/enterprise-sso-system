01 The deadline already started

Every byte encrypted today with classical TLS, RSA, or P-256 is being archived by adversaries with state-level budgets. Harvest now, decrypt later. The U.S. NSA fixed the migration deadlines years ago. The major SSO vendors do not yet have a generally-available answer for identity protocols.

NSA CNSA 2.0 2030 / 2033

  • Software and firmware that signs or distributes new code: full PQC by 2030.
  • New National Security Systems start 2027, complete migration by 2033.
  • Mandates ML-KEM (FIPS 203), ML-DSA (FIPS 204), AES-256, SHA-384/512.

OMB & CISA In effect

  • OMB M-23-02: agencies must inventory cryptography and migrate to PQC, with annual reports.
  • OMB M-22-09: phishing-resistant MFA mandated across federal agencies.
  • NSM-10: NSS migration to quantum-resistant cryptography.

The vendor gap Unfilled

  • Microsoft, Okta, Google, Ping: no GA post-quantum identity protocol as of 2026.
  • TLS-layer PQ (Chrome, SymCrypt) does not protect WebAuthn, OPAQUE, SAML, or token signing.
  • Window of advantage: 2 to 4 years before the incumbents catch up.

06 Cost to certify · the real numbers

Two name corrections first, because they show up in many RFPs:
  • “CNSA Level 5” is not a real standard. CNSA 2.0 is an NSA-mandated algorithm suite (ML-KEM, ML-DSA, AES-256, SHA-384/512), not a tiered level. Buyers usually mean either CNSA-2.0 alignment or Common Criteria EAL5+.
  • “FIPS 103” does not exist. FIPS 140-3 is the current NIST cryptographic module validation; it superseded FIPS 140-2 in 2019, and no new FIPS 140-2 submissions have been accepted since 2022.

09 Honest gaps · what we do not ship today

Regulated buyers prefer a vendor who lists the gaps over one who hides them. Every item below has a matching code path and a 12 to 24-month plan. Nothing is silently broken; every fail-closed default keeps the system safe in the meantime.

Hardware roots SW today

HSM / TPM / SGX / SEV-SNP backends are typed and fail-closed; default deployment uses software simulation through ProductionKeySource. Plug into a customer Cloud HSM partition to flip to physical roots. FIPS 140-3 module is NotSubmitted · budgeted for Year 1 of the cert ladder above.

MDM device posture Stub adapters

Intune, Jamf, and Workspace ONE adapters return NotImplemented · deliberately fail-closed, so any Tier-1 conditional access denies. Wires up cleanly when a tenant supplies API credentials. Not appropriate to claim “device compliance enforced via MDM” until then.

STIG depth 10 + 300 templated

Ten DISA STIG controls are hand-coded and substantive (session timeout, idle lockout, password policy, FIPS mode, audit logging). The remaining 300 are templated derivations · correct but shallow. A real DISA review will credit only the kernel until each family has hand-written checks.

PITR streaming Plan only

pitr-store computes the replay plan given base backups + WAL segments + target timestamp. The actual pg_basebackup + archive_command wiring lives in PostgreSQL config, not in this crate. No RPO / RTO numbers are encoded in code yet · whatever the operator configures is what runs.

Tenant onboarding SQL only

Schema is fully multi-tenant with row-level security and per-tenant compliance regimes. There is no API endpoint to create tenants · provisioning is a SQL INSERT + admin restart to refresh the in-memory ClientRegistry. Self-serve wizard is on the H2 2026 roadmap.

Unwired CAT-J crates Library only

ldap-connector, logout-bch, jit-access, account-linking, offline-jwt compile and pass their own tests but no production binary references them yet. Ship-blocking: integrating each into admin/ is roughly 1 to 2 weeks of work apiece.
