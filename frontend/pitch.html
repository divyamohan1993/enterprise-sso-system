The Problem

  • 01 Authentication systems are the #1 attack vector in modern breaches
  • 02 Well-resourced adversaries can compromise any single server or certificate authority
  • 03 Passwords stored on servers — even hashed — are ticking time bombs
  • 04 To our knowledge, no publicly documented system combines post-quantum + threshold signing + OPAQUE
03 / 10

The Solution

An SSO system where:

  • → The server never sees your password (OPAQUE)
  • → No single server can forge tokens (FROST threshold)
  • → Quantum computers can't break it (ML-KEM + ML-DSA)
  • → Stolen tokens die in seconds (ratcheting sessions)
04 / 10

Value Proposition

“An SSO system that assumes everything is compromised — and still works.”

Military Government Critical Infrastructure Financial Institutions Healthcare
05 / 10

Underlying Magic

  • 01 FROST threshold signing — 3-of-5 signers required
  • 02 OPAQUE (RFC 9807) — server-blind authentication
  • 03 ML-KEM-768 + ML-DSA-65 — post-quantum key exchange & signatures
  • 04 HKDF-SHA512 ratcheting — per-epoch forward secrecy
  • 05 BFT audit chain — tamper-proof logging with hash chains
