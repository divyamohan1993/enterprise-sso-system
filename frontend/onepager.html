01 Use case · why this matters now Why now

Identity for buyers whose regulator has named a 2030 cryptographic deadline. Defence, banking, healthcare, critical infrastructure, sovereign cloud. Microsoft, Okta, Google, Ping have no generally-available post-quantum identity protocol in 2026. The window of advantage is two to four years.

  • NSA CNSA 2.0 mandates ML-KEM (FIPS 203) and ML-DSA (FIPS 204) for new National Security Systems by 2027 start, 2033 complete.
  • OMB M-23-02 requires annual cryptographic inventory and post-quantum migration plan from every federal agency.
  • OMB M-22-09 mandates phishing-resistant MFA across the entire US federal civilian government.
  • Harvest now, decrypt later: any data with greater-than-ten-year confidentiality is already exposed. Healthcare, intelligence, IP filings, M&A, legal hold.
  • Closest analog systems are partial: Apple has Key Transparency at scale, Signal has post-quantum messaging, Estonia has classical threshold signatures. None bundle all four into identity.

02 Deployment · one bash command 30 minutes

One command. Twenty-one VMs. Three availability zones. Active-active. Under thirty minutes from empty cloud project to live OIDC issuer with FROST 3-of-5 token signing.

VPC + IAP→ Shielded VMs→ Key Ceremony→ FROST DKG→ Self-Signed CA→ Systemd Start→ Health Check
  • ./auto_provision.sh --project P --region asia-south2 --nodes 5 · one-shot provisioning.
  • Containers: distroless static-PIE, UID 65534, RELRO/NOW, cosign-signed, CycloneDX 1.5 SBOM emitted from cargo metadata.
  • Kubernetes alternative: 3-replica Patroni Postgres, 3-node etcd, External Secrets Operator + Vault.
  • Migrations: 11 numbered SQL files (tenancy, encrypted PII, audit envelope encryption, DPoP replay cache).
  • Recovery: gateway managed instance group autoscales 1 to 50 on 60% CPU. heal.sh swaps a binary and restarts.
03 Where we deploy · sovereign-ready GCP · Portable

Google Cloud first. Portable to AWS, Azure, on-premise, and air-gapped enclaves. India sovereignty by construction: deployable in Delhi, Mumbai, Hyderabad. DPDP Act 2023 + GDPR + CERT-In hooks built in.

  • GCP regions: asia-south2 (Delhi, live), asia-south1 (Mumbai, script default). Single-region multi-AZ active-active. No cross-region DR yet.
  • 21-VM topology: gateway, orchestrator, opaque, tss, ratchet, verifier, audit (11-node BFT), kt, risk, admin. TSS spread across -a / -b / -c AZs.
  • GCP services touched: Compute (Shielded + AMD SEV-SNP Confidential), VPC + Cloud NAT + IAP, Cloud HSM + KMS + Secret Manager, Cloud Armor, External TCP LB, Binary Authorization, VPC-SC, Access Transparency, Security Command Center Premium.
  • Not used: GKE, Cloud Run, Cloud SQL, Spanner, Pub/Sub. Postgres is self-managed for Confidential VM + dm-verity + PKCS#11 HSM access.
  • Air-gap path: offline-jwt validates against pinned trust stores with CRDT-merged replay caches. Cleared-facility deployable.
