# MILNET SDKs (J15)

Generated client libraries for the MILNET SSO system. Each language stub is
populated by running `openapi-generator-cli` against the OpenAPI document
exported from `authsrv` at build time.

## Languages

- `go/` — generated by `openapi-generator-cli generate -g go`
- `python/` — `-g python`
- `dotnet/` — `-g csharp-netcore`
- `java/` — `-g java`
- `js/` — `-g typescript-fetch`

## Generation

The OpenAPI spec is produced by `authsrv` via the introspection of its
`router()`. The generation pipeline is owned by CI; see
`.github/workflows/sdks.yml` (added in a follow-on PR).

## Why scaffolds only

Per the J15 work-item: "At minimum scaffold + CI to generate." The full
generated client trees are reproducible from the OpenAPI document and are
therefore tracked as build outputs, not source.
